# ARM 汇编基础

## 1. GNU 汇编语法

在使用 GCC 交叉编译器编写 ARM 汇编时，需要遵循 GNU 汇编语法。GNU 汇编文件通常由一行一条语句组成，基本格式如下：

```asm
label: instruction @ comment
```

- `label`：标号，表示地址位置，可用于跳转或表示数据地址。
- `instruction`：指令或伪指令。
- `comment`：注释内容，常用 `@`，也可以使用 `/* ... */`。

示例：

```asm
add:
    movs r0, #0x12    @ r0 = 0x12
```

### 1.1 基本规则

- 指令、伪指令、寄存器名可以全部大写，也可以全部小写，但不要混用。
- 以 `:` 结尾的标识符会被识别为标号。
- `_start` 常作为默认入口标号，也可以在链接脚本中通过 `ENTRY` 指定其他入口。

### 1.2 常见段

可以使用 `.section` 定义段，也可以直接使用 GNU 汇编预定义的常见段名。

| 段名 | 说明 |
| --- | --- |
| `.text` | 代码段 |
| `.data` | 已初始化数据段 |
| `.bss` | 未初始化数据段 |
| `.rodata` | 只读数据段 |

示例：

```asm
.section .testsection
```

### 1.3 常见伪操作

| 伪操作 | 说明 |
| --- | --- |
| `.byte` | 定义单字节数据 |
| `.short` | 定义半字数据 |
| `.long` | 定义 4 字节数据 |
| `.equ` | 定义常量 |
| `.align` | 数据对齐 |
| `.end` | 源文件结束 |
| `.global` | 定义全局符号 |

示例：

```asm
.global _start
.equ num, 0x12
.align 4
```

## 2. 汇编函数

ARM GNU 汇编中的函数通常以标号开头，函数体结束后使用 `bx lr` 返回。

```asm
my_func:
    mov r0, r1
    bx lr
```

说明：

- `bx lr` 是最常见的返回方式。
- 如果函数不需要返回，也可以直接跳转到其他代码路径。

## 3. 常用指令

### 3.1 数据传输指令

常用的数据传输指令有 `mov`、`mrs` 和 `msr`。

| 指令 | 全称 | 作用 |
| --- | --- | --- |
| `mov` | Move | 在通用寄存器之间传递数据，或将立即数加载到寄存器 |
| `mrs` | Move from Special register | 读取特殊寄存器到通用寄存器 |
| `msr` | Move to Special register | 将通用寄存器的数据写入特殊寄存器 |

示例：

```asm
mov r0, r1            @ r0 = r1（寄存器到寄存器）

mov r2, #0x12         @ mov 立即数用法：r2 = 0x12
mov r3, #120          @ mov 立即数用法：r3 = 120

mrs r0, cpsr          @ r0 = cpsr
msr cpsr, r0          @ cpsr = r0
```

补充：当立即数较大、不能直接由 `mov` 编码时，常用 `movw/movt` 或 `ldr rd, =imm`。

```asm
movw r4, #0x5678      @ 低 16 位
movt r4, #0x1234      @ 高 16 位，最终 r4 = 0x12345678
```

### 3.2 存储器访问指令

ARM 在访问外设寄存器或内存数据时，通常使用 `ldr`（Load Register）和 `str`（Store Register）。

| 指令 | 作用 |
| --- | --- |
| `ldr rd, [rn, #offset]` | 从地址 `rn + offset` 读取数据到 `rd` |
| `str rd, [rn, #offset]` | 将 `rd` 中的数据写入地址 `rn + offset` |
| `ldrb rd, [rn, #offset]` | 按字节读取（8 位） |
| `strb rd, [rn, #offset]` | 按字节写入（8 位） |
| `ldrh rd, [rn, #offset]` | 按半字读取（16 位） |
| `strh rd, [rn, #offset]` | 按半字写入（16 位） |
| `ldrsb rd, [rn, #offset]` | 按字节读取并符号扩展 |
| `ldrsh rd, [rn, #offset]` | 按半字读取并符号扩展 |
| `ldr rd, [rn, rm]` | 用寄存器作偏移读取 |
| `str rd, [rn, rm]` | 用寄存器作偏移写入 |
| `ldr rd, [rn, rm, LSL #n]` | 寄存器偏移加移位操作 |
| `ldr rd, [rn, #offset]!` | 预索引寻址，执行后更新 `rn` |
| `str rd, [rn, #offset]!` | 预索引寻址，执行后更新 `rn` |
| `ldr rd, [rn], #offset` | 后索引寻址，执行后更新 `rn` |
| `str rd, [rn], #offset` | 后索引寻址，执行后更新 `rn` |

说明：

- `ldr` 还可以用来加载立即数，这时要写成 `ldr r0, =0x12345678`，不是 `#`。
- `ldr` 和 `str` 默认按字访问；按字节和半字访问时可使用 `ldrb`、`strb`、`ldrh`、`strh`。

`=...` 写法整理：

| 写法 | 是否正确 | 说明 |
| --- | --- | --- |
| `ldr rd, =imm_or_addr` | 正确 | 伪指令，用于把常量（数值或地址）装入寄存器 |
| `str rd, =imm_or_addr` | 不正确 | `str` 没有这种语法，`str` 的目标必须是内存地址表达式（如 `[rn, #offset]`） |

如果你想表达“把 `rd` 写到某个固定地址”，正确步骤是先装地址，再 `str`：

```asm
ldr r0, =0x0209c004   @ 先把目标地址装入寄存器
str r1, [r0]          @ 再把 r1 写入该地址
```

示例：

```asm
ldr r0, =0x0209c004   @ 寄存器地址
ldr r1, [r0]          @ 读取地址中的值

ldr r0, =0x0209c004   @ 寄存器地址
ldr r1, =0x20000002   @ 要写入的值
str r1, [r0]          @ 写入地址
```

### 3.3 压栈和出栈指令

在函数调用或中断处理中，常常需要先保存现场，再恢复现场。常用指令是 `push` 和 `pop`。

| 指令 | 作用 |
| --- | --- |
| `push {reg list}` | 将寄存器列表压入栈中 |
| `pop {reg list}` | 从栈中恢复寄存器列表 |

示例：

```asm
push {r0-r3, r12, lr}

@ 这里执行函数或中断处理

pop {r0-r3, r12, lr}
```

`push` 和 `pop` 也可以看作 `stmfd sp!` 和 `ldmfd sp!` 的常用写法。

```asm
stmfd sp!, {r0-r3, r12, lr}
ldmfd sp!, {r0-r3, r12, lr}
```

#### 3.3.1 STMFD 和 LDMFD 详细说明

`stmfd` 和 `ldmfd` 是 ARM 的多寄存器存取指令，特别适合做现场保护和恢复。

- `stmfd sp!, {寄存器列表}`：把寄存器批量压栈。
- `ldmfd sp!, {寄存器列表}`：把寄存器批量出栈。

在常见的 Full Descending（满递减）栈模型下，它们和 `push`/`pop` 的关系如下：

| 常用写法 | 等价写法 |
| --- | --- |
| `push {r0, r1, lr}` | `stmfd sp!, {r0, r1, lr}` |
| `pop {r0, r1, lr}` | `ldmfd sp!, {r0, r1, lr}` |

其中：

- `sp!` 中的 `!` 表示写回（write-back），执行后会自动更新 `sp`。
- `fd` 表示 Full Descending：栈向低地址增长，`sp` 指向当前栈顶有效数据。

关于 `!` 的补充理解：

- 在 `stmfd sp!, {r0-r3}` 中，`!` 的含义是“本次压栈结束后，把更新后的地址写回 `sp`”。
- 如果写成 `stmfd sp, {r0-r3}`（没有 `!`），数据仍会被写入内存，但 `sp` 本身不会更新。
- 因此在函数现场保护场景里，通常必须使用 `sp!`；否则后续出栈会和预期栈顶不一致，容易导致现场恢复错误。

#### 3.3.2 批量操作寄存器的规则

`stmfd/ldmfd` 一次可以处理多个寄存器，比单条 `str/ldr` 更高效，也更不容易漏寄存器。

核心规则（非常重要）：

- 无论在 `{}` 里写什么顺序，汇编器和硬件都会按寄存器编号重新排序。
- 编号小的寄存器，永远对应内存中更低的地址。
- 这也是为什么建议始终用升序书写寄存器列表，和实际存取顺序一致。

使用要点：

- 寄存器列表用花括号表示，如 `{r0-r3, r12, lr}`。
- 列表支持范围写法，如 `r0-r7`。
- 虽然可以写成乱序列表，但建议按编号升序书写，便于阅读和维护。
- 出栈时的寄存器列表应与压栈完全一致，否则容易破坏现场。

压栈流程示例（`push {r0-r3, r12}` 或 `stmfd sp!, {r0-r3, r12}`）：

1. 硬件先计算总大小：5 个寄存器，共 20 字节。
2. `sp` 先向低地址移动 20 字节。
3. 按“低编号对应低地址”的原则写入：
    - `r0` 写入最低地址（即当前 `sp` 指向处）。
    - `r1`、`r2`、`r3` 写入后续更高地址。
    - `r12`（编号最大）写入这组数据中的最高地址。

出栈流程示例（`pop {r0-r3, r12}` 或 `ldmfd sp!, {r0-r3, r12}`）：

1. 硬件知道要恢复 5 个寄存器。
2. 从当前 `sp` 指向的最低地址开始读取给 `r0`。
3. 地址每次加 4，依次恢复 `r1`、`r2`、`r3`。
4. 最高地址的数据恢复给 `r12`。
5. 最后 `sp` 回到压栈前的位置。

可以把它理解为：`ldmfd` 基本就是 `stmfd` 的镜像操作。

示例（批量压栈/出栈）：

```asm
stmfd sp!, {r4-r11, lr}   @ 批量保存被调用者保存寄存器和返回地址

@ 函数主体

ldmfd sp!, {r4-r11, lr}   @ 批量恢复
bx lr
```

#### 3.3.3 案例 1：替代 PUSH/POP 的函数模板

下面两段代码语义等价。

写法 A（常见）：

```asm
push {r4-r7, lr}

@ do something

pop {r4-r7, lr}
bx lr
```

写法 B（等价底层写法）：

```asm
stmfd sp!, {r4-r7, lr}

@ do something

ldmfd sp!, {r4-r7, lr}
bx lr
```

结论：

- 在 ARM 文档、启动代码、中断入口代码中，经常直接看到 `stmfd/ldmfd`。
- 在日常手写函数时，很多人更习惯 `push/pop`，可读性更直观。

#### 3.3.4 案例 2：中断服务中的现场保护

IRQ/FIQ 处理中，常用 `stmfd/ldmfd` 成组保存上下文。

```asm
irq_handler:
    stmfd sp!, {r0-r3, r12, lr}   @ 保存最常用现场

    bl system_irqhandler           @ 调用 C 中断处理函数

    ldmfd sp!, {r0-r3, r12, lr}   @ 恢复现场
    subs pc, lr, #4               @ 从 IRQ 返回（示例写法）
```

说明：

- 这种写法能减少漏保存寄存器的风险。
- 具体保存哪些寄存器取决于 ABI、编译器约定和中断处理流程。
- 返回指令形式（如 `subs pc, lr, #4`）与处理器模式、异常类型有关，工程里要统一规范。

### 3.4 跳转指令

常见跳转方式有 `b`、`bl`、`bx` 和 `blx`。

| 指令 | 作用 |
| --- | --- |
| `b <label>` | 直接跳转到标号 |
| `bl <label>` | 跳转到标号，并把返回地址保存到 `lr` |
| `bx <rm>` | 跳转到寄存器指定地址，并切换指令集状态 |
| `blx <rm>` | 结合 `bl` 和 `bx` 的特点，支持间接调用和指令集切换 |

说明：

- `b` 适合不需要返回的跳转。
- `bl` 适合函数调用。
- `bx lr` 常用于函数返回。

示例：

```asm
_start:
    ldr sp, =0x80200000
    b main
```

如果需要从汇编调用 C 函数，通常使用 `bl`：

```asm
bl system_irqhandler
```

### 3.5 算术运算指令

常用的算术运算指令如下。

| 指令 | 作用 |
| --- | --- |
| `add` | 加法 |
| `adc` | 带进位加法 |
| `sub` | 减法 |
| `sbc` | 带借位减法 |
| `mul` | 乘法 |
| `udiv` | 无符号除法 |
| `sdiv` | 有符号除法 |

示例：

```asm
add r0, r1, r2
sub r0, r1, #1
mul r0, r1, r2

adc r0, r1, r2      @ 带进位加法：r0 = r1 + r2 + C
sbc r0, r1, r2      @ 带借位减法：r0 = r1 - r2 - (1 - C)

udiv r0, r1, r2     @ 无符号除法：r0 = (uint32_t)r1 / (uint32_t)r2
sdiv r0, r1, r2     @ 有符号除法：r0 = (int32_t)r1 / (int32_t)r2
```

关于 `adc` 和 `sbc` 里的“位从哪来”：

- 这个位来自状态寄存器 CPSR 中的 C 标志位（Carry flag）。
- C 标志位通常由前一条会更新标志位的指令产生，比如 `adds`、`subs`、`cmp`、`cmn`、`adcs`、`sbcs`。
- `adc` 会把当前 C 当作额外的加 1（或加 0）参与运算。
- `sbc` 使用的是“非借位”语义：C=1 表示上一条减法没有借位，C=0 表示发生了借位，所以公式是 `r1 - r2 - (1 - C)`。

常见配合写法：

```asm
adds r4, r4, #1     @ 先做低位加法并更新 C
adc  r5, r5, #0     @ 再把 C 加到高位，做多精度加法

cmp  r6, r7         @ 比较后更新 C（无借位则 C=1）
sbc  r0, r1, r2     @ 按 C 状态做带借位减法
```

### 3.6 逻辑运算指令

常用的逻辑运算指令如下。

| 指令 | 作用 |
| --- | --- |
| `and` | 按位与 |
| `orr` | 按位或 |
| `bic` | 位清除 |
| `eor` | 按位异或 |
| `orn` | 按位或非 |

示例：

```asm
and r0, r1, r2      @ r0 = r1 & r2，按位与
orr r0, r1, #0x1    @ r0 = r1 | 0x1，按位或
bic r0, r1, #0x3    @ r0 = r1 & ~0x3，清除低 2 位
eor r0, r1, r2      @ r0 = r1 ^ r2，按位异或
```
